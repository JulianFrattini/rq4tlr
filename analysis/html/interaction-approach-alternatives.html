Interaction: Approach x Contains Alternatives

Anonymous

2025-08-30

This notebook contains the analysis of the potential interaction effect between TLR approach (approach) and Contains Alternatives (alternative) on the outcome (here: f2). The analysis provides insight into the hypothesis that some approaches perform differently well for different levels of alternatives contained in a use case.

Data Loading

First, load the data for the observational analysis. The data set is procured by the data-preparation.Rmd notebook. This notebook assembles and casts the data appropriately and stores the results in the rq4tlr-data.csv file.

df <- read.csv('./../data/rq4tlr-data.csv')

Scale all numeric variables to make their coefficient distributions easier to sample from.

Analysis

The analysis of the data employs a regression model in a Bayesian framework. The utilized formula only assumes an intercept (i.e., a baseline), an offset per dataset, and the interaction effect between approach and the current variable of interest.

f <- F2 ~ 0 + Intercept + Dataset + Approach*alternative
get_prior(f, data=df, family=zero_one_inflated_beta)

Given this formula, select uninformative distributions for each of the coefficient distributions.

p1 <- c(
  prior(normal(0, 1), class = b, coef = "Intercept"),
  prior(normal(0, 0.5), class = b),
  prior(gamma(4, 1), class = phi),
  prior(beta(7, 2), class = coi), 
  prior(beta(5, 2), class = zoi) 
)

To confirm the eligibility of the prior distributions, perform a prior predictive check (skipped in the version produced by knitr).

m.prior <-
  brm(data = df, family = zero_one_inflated_beta, f, prior = p1,
    iter = 4000, warmup = 1000, chains = 4, cores = 4,
    seed = 4, sample_prior="only",
    file = "../fits/int.f2.approach.alternative.prior"
  )

brms::pp_check(m.prior, ndraws=100)

Given the eligibility of the priors, train the full model by updating the coefficient distributions in light of the observed data.

m <-
  brm(data = df, family = zero_one_inflated_beta, f, prior = p1,
    iter = 4000, warmup = 1000, chains = 4, cores = 4, seed = 4,
    file = "../fits/int.f2.approach.alternative"
  )

brms::pp_check(m, ndraws=100)

The posterior predictive check confirms that the model trained properly.

More contextualized insights can be derived by plotting the conditional effect.

conditional_effects(m, effects="alternative:Approach", 
                    prob = 0.5, method="posterior_epred")

This invites the following interpretations:

  • Language model-based approaches again seem to benefit or are not as much affected by alternatives and things misplaced in the use cases.
  • VSM and LSI seem to be not able to grasp the alternatives as well if they are misplaced.
