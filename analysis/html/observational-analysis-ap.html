This notebook contains the observational analysis of the impact of requirements quality (RQ) on trace link recovery (TLR).

Data Loading

First, load the data for the observational analysis. The data set is procured by the data-preparation.Rmd notebook. This notebook assembles and casts the data appropriately and stores the results in the rq4tlr-dtata.csv file.

df <- read.csv('./../data/rq4tlr-data.csv')

Analysis

The analysis of the data employs a regression model in a Bayesian framework.

Distribution

The three outcome variables Precision, Recall, and F1 are all bounded \(\in [0, 1]\). This makes a Beta distribution an eligible target distribution of the response variables. To determine which variant of the Beta distribution family can be chosen, we visualize the raw distribution.

df %>% 
  select(AP) %>% 
  ggplot(aes(x = AP)) +
    geom_histogram(bins = 30)

A base Beta distribution would require that all values are bounded within \(]0; 1[\). The observed values for the three outcome variables show a clear zero- and one-inflation. Hence, we chose the zero_one_inflated_beta distribution from the brms library.

Formula

Next, we define regression formulae that put the outcome variable of interest in relation to the predictors with which we assume a causal relationship.

f <- AP ~ 0 + Intercept + Dataset + sencomplexity + maxsteplength + size + inconsistentloa + uidesign + clarification + alternative + coherence + happy + lostfr + anaphora + nonnounstart + passive + negation + outlinks + maxinlinks

The 0 + Intercept term instead of 1 + or even complete omission stems from the fact that the regression formula contains at least one predictor that is not centered around 0.

We select the following priors:

p1 <- c(
  prior(normal(0, 1), class = b, coef = "Intercept"),
  prior(normal(0, 0.5), class = b),
  prior(gamma(4, 1), class = phi),
  prior(beta(6, 2), class = coi), 
  prior(beta(4, 2), class = zoi) 
)

Prior Predictive Check

The prior predictive check helps determine the eligibility of the priors.

m1.prior <-
  brm(data = df, family = zero_one_inflated_beta, f, prior = p1,
    iter = 4000, warmup = 1000, chains = 4, cores = 4,
    seed = 4, sample_prior="only",
    file = "../fits/obs.ap.prior"
  )

In our particular case, the large number of predictors will, however, push the accumulated value of \(\mu\) in the Beta-distribution to the to extremes.

brms::pp_check(m1.prior, ndraws=100)
## Warning: Computation failed in `stat_density()`

The figure shows extreme concentrations of expected values \(y_{rep}\) on the borders of the output space (at 0 and 1). Instead, we put more emphasis on the posterior predictive check.

Training

Assuming the priors are reliable, we train the model by updating the coefficient parameters with the data df.

m1 <-
  brm(data = df, family = zero_one_inflated_beta, f, prior = p1,
    iter = 4000, warmup = 1000, chains = 4, cores = 4, seed = 4,
    file = "../fits/obs.ap"
  )

To check whether the training completed successfully, we inspect the posterior predictive check.

brms::pp_check(m1, ndraws=100)

The output confirms that the model parameters have been updated properly and that the predicted outcomes \(y_{rep}\) are close the the actual outcome \(y\).
