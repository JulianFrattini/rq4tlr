Analysis

The analysis of the data employs a regression model in a Bayesian framework.

Distribution

The outcome variable is bounded within \(\in [0, 1]\). This makes a Beta distribution an eligible target distribution for the response variables. To determine which variant of the Beta distribution family can be chosen, we visualize the raw distribution.

df %>% 
  select(Recall) %>% 
  ggplot(aes(x = Recall)) +
    geom_histogram(bins = 30)

A base Beta distribution would require that all values are bounded within \(]0; 1[\). The observed values for the outcome variable show a clear zero- and one-inflation. Hence, we chose the zero_one_inflated_beta distribution from the brms library.

Formula

Next, we define a regression formula that puts the outcome variable of interest in relation to the predictors with which we assume a causal relationship.

f <- Recall ~ 0 + Intercept + Dataset + Approach + sencomplexity + maxsteplength + size + inconsistentloa + uidesign + clarification + alternative + coherence + happy + lostfr + anaphora + nonnounstart + passive + negation + outlinks + maxinlinks

The 0 + Intercept term instead of 1 + or even complete omission stems from the fact that the regression formula contains at least one predictor that is not centered around 0.

We select uninformative prior distributions for all coefficients to allow the model to learn any effect from the data.

p1 <- c(
  prior(normal(0, 1), class = b, coef = "Intercept"),
  prior(normal(0, 0.2), class = b),
  prior(gamma(4, 1), class = phi),
  prior(beta(6, 2), class = coi), 
  prior(beta(6, 2), class = zoi) 
)

Prior Predictive Check

The prior predictive check helps determine the eligibility of the priors.

m1.prior <-
  brm(data = df, family = zero_one_inflated_beta, f, prior = p1,
    iter = 4000, warmup = 1000, chains = 4, cores = 4,
    seed = 4, sample_prior="only",
    file = "../fits/obs.recall.prior"
  )

In our particular case, the large number of predictors will, however, push the accumulated value of \(\mu\) in the Beta-distribution to the to extremes.

brms::pp_check(m1.prior, ndraws=100)

The figure shows extreme concentrations of expected values \(y_{rep}\) on the borders of the output space (at 0 and 1). This is expected due to the large number of predictors. Hence, we put more emphasis on the posterior predictive check.

Training

Assuming the priors are reliable, we train the model by updating the coefficient parameters with the data df.

m1 <-
  brm(data = df, family = zero_one_inflated_beta, f, prior = p1,
    iter = 4000, warmup = 1000, chains = 4, cores = 4, seed = 4,
    file = "../fits/obs.recall"
  )

To check whether the training completed successfully, we inspect the posterior predictive check.

brms::pp_check(m1, ndraws=100)

The output confirms that the model parameters have been updated properly and that the predicted outcomes \(y_{rep}\) are close the the actual outcome \(y\).

Marginal Effects

For a clearer picture, we can plot the marginal effect of any predictor on the outcome variable. The marginal effect isolates the effect of the predictor on the outcome while keeping all other predictors constant at a representative level.

conditional_effects(m1, effects="sencomplexity", method="posterior_epred")

conditional_effects(m1, effects="coherence", method="posterior_epred")

conditional_effects(m1, effects="outlinks", method="posterior_epred")

conditional_effects(m1, effects="Approach", method="posterior_epred")

Particularly, we extract the marginal values for the effect on Contains Alternatives (alternative), Contains Anaphora (anaphora), and Tangled Requirements (outlinks) to visualize them (side-by-side with all other metrics).

c.eff.alternatives <- conditional_effects(m1, effects="alternative", method="posterior_epred")

c.eff.alternatives$alternative %>% 
  select(alternative, estimate__, lower__, upper__) %>% 
  rename(all_of(c(
    mean = "estimate__", 
    lower = "lower__", 
    upper = "upper__"
  ))) %>% 
  mutate(metric = "Recall") %>% 
  write_csv(file = "../data/estimates/marginal-alternatives-recall.csv")
c.eff.anaphora <- conditional_effects(m1, effects="anaphora", method="posterior_epred")

c.eff.anaphora$anaphora %>%

When exporting the marginal effects of the outlinks variable, we can directly revert the centering and scaling from the beginning (with \(\mu=3.15\) and \(\sigma=2.46\)).

c.eff.outlinks <- conditional_effects(m1, effects="outlinks", method="posterior_epred")

c.eff.outlinks$outlinks %>% 
  select(outlinks, estimate__, lower__, upper__) %>% 
  rename(all_of(c(
    mean = "estimate__", 
    lower = "lower__", 
    upper = "upper__"
  ))) %>% 
  mutate(
    outlinks = outlinks*2.46 + 3.15,
    metric = "Recall"
    ) %>% 
  write_csv(file = "../data/estimates/marginal-outlinks-recall.csv")

Store Data

Finally, we store the estimates of the posterior distributions in a new data frame and export it. This allows the estimate-visualization script to generate one figure of the posterior distributions from all response variables.

summary(m1)$fixed %>% 
  select(Estimate, Est.Error) %>% 
  rename(all_of(c(
    mean = "Estimate", 
    var = "Est.Error"))) %>% 
  rownames_to_column("coef") %>% 
  write_csv(file = "../data/estimates/obs-recall.csv")
